导数的定义

设函数 $f(x)$ 在点 $x_0$ 的某个邻域内有定义，当自变量 $x$ 在 $x_0$ 处有增量 $\Delta x$，点 $x + \Delta x$ 也在该邻域时，对应的函数取得增量 $\Delta y$。如果 $\Delta y$ 与 $\Delta x$ 之比当 $\Delta x \longrightarrow 0$ 时，极限存在，则称函数 $f(x)$ 在 $x_0$ 处可导（仅在 $x_0$ 这一点处，并不保证在所有点处），并称这个极限为函数 $f(x)$ 在点 $x_0$ 处的导数,记为 $f’(x_0)$ ，即：

对于一个函数来说,可导必定连续,连续不一定可导(没有定义)

无穷小与无穷大

Notes!:无穷小和无穷大是一种变量,不是具体的数值。

  • 无穷小以数0为极限的变量，无限接近于0
  • 无穷大是超出任何有界值的数值

  • 无穷小的定义

如果函数 $f(x)$ 当 $x \to x_0$ (或 $x \to \infty$)时的极限为 $0$ ,那么称函数 $f(x)$ 为当 $x \to x_0$ (或 $x \to \infty$)时的无穷小。

无穷大的定义

如果当 $x \to x_0$ (或 $x \to \infty$)时,对应的函数值的绝对值 $|f(x)|$ 可以大于预先指定的任何很大的正数 $M$ ,就称函数 $f(x)$ 是当 $x \to x_0$ (或 $x \to \infty$)时的无穷大。

等价无穷小

非常重要！！！！

极限的运算法则

适用前提！！！拆开后形成的函数均存在极限！！

定理

  1. 有限个无穷小之和是无穷小

  2. 有界函数与无穷小的乘积是无穷小

  3. 若 $\lim f(x) =A,\lim g(x) = B$ 那么

    • $\lim [f(x) \pm g(x)]=\lim f(x) \pm \lim g(x)=A\pm B$
    • $\lim [f(x) \cdot g(x)]=\lim f(x) \cdot \lim g(x)=A\cdot B$
    • $\lim \frac{f(x)}{g(x)} = \frac{\lim f(x)}{\lim g(x)} = \frac{A}{B}[B \ne 0]$
  4. 若 $\varphi (x) \ge \psi (x)$,$\lim \varphi (x) =A ,\lim \psi (x)=B$,则$A \ge B$(用$f(x) = \varphi (x) - \psi (x)$加上定理3推导)
  5. 复合函数的极限运算法则

两个重要极限

无穷小的比较

  • 如果 $\lim \frac{\beta }{\alpha } = 0$ ，那么 $\beta$ 是比 $\alpha$ 高阶的无穷小，记作 $\beta = o(\alpha)$
  • 如果 $\lim \frac{\beta }{\alpha } = \infty$ ，那么 $\beta$ 是比 $\alpha$ 低阶的无穷小。
  • 如果 $\lim \frac{\beta }{\alpha } = C \ne 0$ ，那么 $\beta$ 和 $\alpha$ 是同价无穷小。
  • 如果 $\lim \frac{\beta }{\alpha ^ k} = C \ne 0$ ，那么 $\beta$ 是关于 $\alpha$ 的 $k$ 价无穷小。
  • 如果 $\lim \frac{\beta }{\alpha } = 1$ ，那么 $\beta$ 和 $\alpha$ 是等价无穷小。

函数的连续性和间断点

若 $f(x)$ 在点 $x_0$ 处连续，则需满足下面三个条件

  • $f(x)$ 在 $x_0$ 处有意义
  • $f(x)$ 在 $x_0$ 处的极限存在
  • $\lim_{x \to x_0} f(x) = f(x_0)$

其中第三条有等价形式

间断点的定义及其分类

定义：

在上述三个(判断函数的连续性)条件中至少一项不成立，则称点 $x_0$ 为函数 $f(x)$ 的不连续点或间断点。

交互题(Interactive)

交互题 是需要选手程序与测评程序交互来完成任务的题目。一类常见的情形是，选手程序向测评程序发出询问，并得到其反馈。

交互题的特殊错误：

  • 选手每一次输出后都需要刷新缓冲区，否则会引起 Idleness limit exceeded 错误。另外，如果题目含多组数据并且程序可以在未读入所有数据前就知道答案，也仍然要读入所有数据，否则同样会因为读入混乱引起 ILE（可以一次提出多次询问，一次接收所有询问的回答）。同时尽量不要使用快读。
  • 如果程序查询次数过多，则在 Codeforces 上会给出 Wrong Answer 的评测结果（不过评测系统会说明 Wrong Answer 的原因），而 UVa 会给出 Protocol Limit Exceeded (PLE) 的评测结果。
  • 如果程序交互格式错误，UVa 会给出 Protocol Violation (PV) 的评测结果。

由于交互题输入输出较为繁琐，所以建议分别封装输入和输出函数。

交互题的特殊性

在所有的输出语句结束后都应该清空缓冲区

可以使用如下语句来清空缓冲区：

  • 对于 C/Cpp: fflush(stdout)
  • 对于 Cpp: std::cout << std::flush
  • 对于 Java: System.out.flush()
  • 对于 Python: stdout.flush()
  • 对于 Pascal: flush(output)

特别的，对于 Cpp 语言,输出换行时使用 std::endl 而非 \n 也能够自动刷新缓冲区
